# Aily PPT 模板生成器（v3）

## 概述

把"模板"和"内容"解耦为**三层抽象**：
- **版式层**：模板的版式、字体、配色、装饰元素
- **特征层**：从多张同模板示例中归纳"共性 zone"（机构标签区/主标题区/正文区/出处区）
- **内容层**：每个 zone 下的"变量"——用户提供新内容

v3 关键能力：
- **多页归纳**：从 3 张同模板示例中提取"4 个共性 zone"
- **逐 run 替换**：模板的"标签"和"内容"分开处理
- **多页输出**：一次性生成 N 页同结构 PPT

## v3 vs v2 升级

| 能力 | v2 | v3 |
|------|----|----|
| 模板归纳 | 单 slide 启发式 | 多 slide 统计归纳 |
| 模板 vs 变量 | 逐 run 规则 | 逐 run + 区域共性统计 |
| 多页生成 | 不支持 | ✅ content.slides 数组 |
| 关键强调格式 | 忽略 | 保留（如"红色加粗"） |
| 区域理解 | 无 | zone-based（机构/标题/正文/出处） |

## 适用场景

- **公司/部门多页汇报模板**：用户提供 3-5 张同模板示例，技能归纳共性，按主题生成多页 PPT
- **行业研究类模板**（如"AI云BMS五看"）：每页结构相似（机构+标题+正文+出处）
- **半结构化模板**：模板中既有固定标签（5W2H/When/Where/编号/小标题）也有变动内容

## 不适用场景

- 用户没有模板，希望从零设计 → 应使用通用 PPT 生成能力
- 飞书 Slide 在线版模板 → 不在本技能范围
- 模板里所有文字都是变量 → 用通用 PPT 生成更合适

## 工作流程（v3）

```
用户提供 源 PPT（含 N 张同模板 slide）
    ↓
Step 1: extract_template_features.py   → template_features.json
                                       （归纳 N 张 slide 的共性 zone）
    ↓
Step 2: detect_template_slots.py       → slot_map.json
                                       （逐 run 判定 template/variable）
    ↓
Step 3: AI 生成 content.json           （每张新内容页的 fill_map + clear_others）
    ↓
Step 4: generate_pptx.py               → 最终 .pptx（多页）
    ↓
Step 5: 用户在 PowerPoint 中二次编辑
```

## 详细步骤

### Step 1: 多页模板特征归纳

```bash
python scripts/extract_template_features.py <template.pptx> template_features.json
```

输出包含：
- `num_slides_analyzed`: 分析的源 slide 数
- `zones`: 归纳出的共性 zone
  - 每个 zone 有：role/size/color/font/position/variable_rules

典型 zone：
- `header_institution`: 顶部机构标签
- `title_main`: 主标题（22pt 加粗）
- `body_left`: 左正文
- `body_right`: 右图表
- `footer_source`: 底部出处/法规

### Step 2: 逐 run slot 检测

```bash
python scripts/detect_template_slots.py <template.pptx> slot_map.json [--mode heuristic|ai-hint]
```

检测每个 run 是 `template`(保留) 还是 `variable`(替换)。

### Step 3: AI 生成内容 JSON

```json
{
  "slides": [
    {
      "use_slide_idx": 0,
      "fill_map": {
        "sh0.p0.r0": "清华大学-电驱安全组（汽车工程系）",
        "sh1.p0.r0": "电驱温度估算",
        "sh2.p0.r0": "核心痛点：温度传感器布点受限...",
        "sh3.p0.r0": "Wang J. et al. ..."
      },
      "clear_others": {
        "sh0.p0": 0,
        "sh1.p0": 0,
        "sh2.p0": 0,
        "sh3.p0": 0
      }
    },
    {
      "use_slide_idx": 1,
      "fill_map": { ... 第 2 页内容 ... }
    }
  ]
}
```

**关键约定**：
- `use_slide_idx` 引用源模板的哪张 slide（0-based）
- `fill_map` 按 path 精确指定每个 run 的内容
- `clear_others` 指定段落清空其他 run（保留指定 run 的样式）

### Step 4: 多页 PPT 生成

```bash
python scripts/generate_pptx.py <template.pptx> slot_map.json content.json output.pptx
```

- 按 `use_slide_idx` 复制源 slide
- 按 fill_map 的 path 精确替换
- 按 clear_others 清空其他 run
- **保留模板所有视觉风格**（字体、字号、颜色、加粗、装饰元素）

### Step 5: 二次编辑

用户在 PowerPoint 中打开 `output.pptx`，可自由修改任意文字、调整位置、更换图片。

### 缓冲规则（v3.8.5v3 压紧版）

| 区段类型 | 缓冲 | 何时 |
|---------|------|------|
| 标题不换行 → 正文 | **0.05"** | 压紧 |
| 段间同字号 | **0.05-0.10"** | 紧贴 |
| 段间异字号（差 ≥ 2pt）| **0.20"** | 适中 |
| 带状装饰 → 正文 | **0.04-0.10"** | 紧贴 OK |
| 换行的段 | **预留 2 行 h** | 主动预留 |
| Quote → Source | **0.20"** | 段间 |
| Page bottom | **0.40"+** | 富余 |
| **段间撞了** | **0.00" 以下** | **绝对禁止** |

### 关键算法函数（v3.8.5v3 必背）

```python
def will_wrap(text, font_pt, box_w):
    """微软雅黑实测系数 (基于 20pt 实际渲染 8 次)"""
    cn = sum(1 for c in text if '一' <= c <= '鿿')
    en = sum(1 for c in text if c.isalnum() and not ('一' <= c <= '鿿'))
    sp = len(text) - cn - en
    est_w = cn * 0.014 * font_pt + en * 0.007 * font_pt + sp * 0.005 * font_pt
    eff = box_w - 0.04
    return est_w > eff * 0.95, est_w, eff

def actual_lines(text, font_pt, box_w):
    """不换行=段落数, 换行=ceil(est_w / eff)"""
    will, est_w, eff = will_wrap(text, font_pt, box_w)
    if not will: return sum(1 for p in text.split("\n") if p.strip())
    return max(1, int(est_w / eff) + 1)

def calc_h_auto(text, font_pt, box_w, line_spacing=1.20, padding=0.05):
    """根据实际行数算 h"""
    lines = actual_lines(text, font_pt, box_w)
    return lines * font_pt * line_spacing / 72.0 + padding * 2
```

### 关键教训（老大 2026-06-30 反馈 6 条）

1. **"换行 = 高度翻倍 = 溢出"** — 标题 36 字 20pt 换行后 h 0.45" 装不下 2 行 → "理"字溢出 → 视觉重叠
2. **"不换行就不浪费"** — 0.30" 缓冲压到 0.05" 是 P0 优先
3. **"算法对 ≠ 视觉对"** — PowerPoint 字体渲染会向上溢出 0.05-0.10"，必须预留
4. **"先看菜点菜，再切锅"** — 按内容类型 + 元素数量动态划分，不硬 4 等分
5. **"原则冲突时优先级要明确"** — 老大钦定 P0=零堆叠，其他都可让步
6. **"老大截图是终极标准"** — 算法对不算赢，视觉对才算赢

## 核心设计原则（架构层，与"原则总览"互补）

1. **三层抽象分离**：模板特征 / 半结构化识别 / 内容填充
2. **统计归纳代替规则**：从 N 张示例中提取共性，而不是硬编码规则
3. **样式继承**：不硬编码颜色/字体/字号——填占位符后由模板版式自动应用
4. **逐 run 精度**：每个 run 单独判定，避免"整段替换"破坏标签
5. **可编辑**：所有内容都是普通占位符文本，用户可在 PowerPoint 中自由修改
6. **无水印**：直接用 python-pptx 操作模板
7. **⭐ 信息密度原则（v3.1 新增）**：参考示例 PPT 的"每页字数"和"信息类型"作为硬约束，禁止生成"低密度大字版"。

## ⭐ 信息密度原则（v3.1 关键约束）

**这是用户在使用本技能时最常被违反的原则。** 任何 PPT 生成都必须先分析**源模板的密度基线**，再决定新内容的密度。

### 7.1 密度基线（参考"AI云BMS-五看 模版"）

| 区域 | 字号 | 字符数（参考） | 信息类型 |
|------|------|---------------|---------|
| 顶部小标签 | 11pt 加粗红色 | 10-20 字符 | 机构/板块名 |
| 主标题区 | 22pt 加粗 + 12-14pt 正文 | 400-600 字符 | 主标题 + 痛点 + 方案 + 成效 + 意义（多段） |
| 左正文 | 9.5-12pt + 11pt 加粗小标题 | 700-900 字符 | 多段密集技术内容（5-6 段小标题+要点） |
| 图 1 | 8.5pt 表格化文字 | 100-150 字符 | 表格/架构图/对比表（带边框） |
| 图 2 | 8.5pt 表格化文字 | 100-150 字符 | KPI 对比/路线图/玩家矩阵 |
| 底部引言 | 11pt + 红色加粗 | 60-100 字符 | 金句（可含红色加粗关键词） |
| 底部出处 | 8.5pt 灰色斜体 | 100-150 字符 | 英文期刊/公司年报引用 |

**每页总字符数参考基线：1300-1800 字符**（v1 低密度版仅 380-450 字符，未达基线）。

### 7.2 信息类型多元化（避免"纯文字堆"）

| 必备类型 | 出现位置 | 示例 |
|---------|---------|------|
| **数字 / KPI** | 主标题/左正文/表格 | "21 个实验→762 万训练点"、"95% 请求 1 秒内响应" |
| **表格** | 图位置或左正文 | 端-边-云 6 组件、玩家矩阵、路线图 |
| **关键概念加粗** | 红色 #FF0000 加粗 | 痛点、关键数据、差异化关键词 |
| **学术引用** | 主标题/出处 | Joule 2024、RSC 2025、GB/T 45678-2025 |
| **行业数据** | 主标题/正文 | "全球 3.4 亿美元 CAGR 19.7%"、"北美 1.634 亿美元" |
| **金句 / 引用** | 底部 | "X 不是 Y，而是 Z" 格式 |
| **图 / 示意图** | 图 1/图 2 | 架构图、矩阵图、路线图（占位） |

**禁止**：单页纯文字堆（无数字、无表格、无引用、无关键加粗）。

### 7.3 字间距与排版（避免"大字稀疏"）

| 反面案例 | 修正方式 |
|---------|---------|
| 22pt 大标题 + 50 字 | 22pt 主标题 + 12-14pt 正文 500 字 |
| 18pt 要点 + 单行 | 11-13pt 要点 + 3-5 行密集排列 |
| 占位符间空一行 | 段间距统一 0.05-0.1 inch |
| 左正文 200 字 | 左正文 700-900 字（5-6 个小标题+要点） |

### 7.4 信息量扩充策略（内容不够时）

如果调研结果达不到 1300 字符基线，按以下优先级扩充：

1. **加引用**：补 Joule / RSC / Nature / 行业报告（每条 50-100 字符）
2. **加数字**：所有"很强"→"高 95%"，"很多"→"PB 级"
3. **加对比表**：把"3 个要点"变成"6 维度对比表"
4. **加行业数据**：市场规模 / CAGR / 装机量 / 出货量
5. **加政策依据**：GB/T 45678-2025 / EU 2023/1542 / ISO 26262

## ⭐ 信息分层原则（v3.2 关键约束）—— 解决"文字堆叠"问题

**这是 v3.1 之后用户反馈的第二个高频问题。** v3.1 解决了"信息密度太低"，但**信息密度高 ≠ 文字堆叠**。真正的标准是 **"高密度 + 强分层"**。

### 7.8 核心问题诊断

| v3.1 的"高密度"做到了 | 但产生了"堆叠"问题 |
|---------------------|------------------|
| 字符数 7300 | 全部挤在 9.5-12pt 一个字号段 |
| 含 5-6 个小标题 | 但跟正文差距仅 2-3pt，视觉分层弱 |
| 主标题 7 段 | 但段间没分隔线、段间距 0 |
| 左正文密集 | 段间 line_spacing 默认 1.0，挤成一片 |

**根因**：v3.1 量化了"信息量"，但没量化"信息层级"。

### 7.9 三级字号梯度（强制）

**单一字号 + 高密度 = 堆叠**。正确做法是 **"大-中-小三档字号"**：

| 层级 | 字号 | 加粗 | 颜色 | 用途 |
|------|------|------|------|------|
| **L1 主标题** | 22pt | ✅ | 黑 | 1-2 段核心句 |
| **L2 小标题** | 14pt | ✅ | 红 #CC0000 | 5-6 个章节小标题（**强制比正文大 2pt+**） |
| **L3 正文** | 11pt | ❌ | 深灰 #333333 | 5-6 段要点 |
| **L4 强调** | 11pt | ✅ | 红 #FF0000 | 关键短语（嵌入 L3 段落中） |

**关键约束**：L2 和 L3 字号差必须 ≥ 2pt，否则视为"分层不足"。

### 7.10 段落分隔与行高（强制）

| 项目 | 强制值 | 修复后效果 |
|------|-------|----------|
| 段落间分隔线 | `---` 横线（每 4-5 段加 1 条） | 视觉分段清晰 |
| `line_spacing` | **≥ 1.4**（python-pptx 中设 1.4） | 段内行间呼吸感 |
| `space_before` | **≥ 4pt**（段前留白） | 段间缓冲 |
| `space_after` | **≥ 2pt**（段后留白） | 段间缓冲 |
| 引用块 `>` | 单独成行 + 左侧缩进 0.1 inch | 金句与正文视觉分离 |

### 7.11 段落结构模板（必查清单）

每张高密度页必须满足：

```

[主标题 L1] 主题（1-2 段）
  ├ [痛点 L2] 用红色加粗关键词

  ├ [成效 L2] 关键数据
```

### 7.12 文字堆叠反面案例 vs 正确做法

| 反面案例 | 正确做法 |

### 7.22 段溢出自动检测算法（v3.4 创新）

```python
def estimate_segment_height(text, font_size_pt, line_spacing=1.3):
    """估算单段在 PPT 里占用的高度"""
    char_width_pt = font_size_pt * 0.85  # 中英文混合
    box_width_pt = 8.00 * 72  # left_body 8 inch
    chars_per_line = int(box_width_pt / char_width_pt)
    n_lines = ceil(len(text) / chars_per_line)
    h = n_lines * font_size_pt * line_spacing + 4 + 2  # +段前段后
    return h
```

**判断公式**：
- 若 `Σ estimate_segment_height(text_i, font_pt) ≤ available_height_pt` → 用此字号
- 否则字号降档：11pt → 10pt → 9pt → 8pt 兜底
- 段数限制：title ≤ 8 段，left_body ≤ 18 段

### 7.23 段溢出自动调整函数（v3.4 强制）

```python
def auto_fit_runs(runs_spec, available_height_pt, max_segments=20):
    """
    1. 限制段数 ≤ max_segments
    2. 动态缩字号：11→10→9→8
    3. 段总高度 ≤ available_height_pt
    """
    # 1. 限制段数
    if len(runs_spec) > max_segments:
        # 中间段合并成 summary
        third = max_segments // 3
        head = runs_spec[:third]
        middle = runs_spec[third:-third]
        tail = runs_spec[-third:]
        # 合并中间
        if len(middle) > 1:
            combined = '｜'.join(t[1] for t in middle if t[0] == 'n')
            middle = [('n', combined[:200] + '...')]
        runs_spec = head + middle + tail

    # 2. 动态缩字号
    for size in [11, 10, 9, 8]:
        total_h = sum(estimate_segment_height(t[1], size) for t in runs_spec)
        if total_h <= available_height_pt:
            return runs_spec, size
    return runs_spec, 8
```

**根因**：`shape.height` 设的太小，文字溢出到下一个 textbox → 视觉"半透明叠印"。

## v3.7.5 真实行高检测公式

```python
def calc_h(text, font_pt, line_spacing=1.20, padding=0.04):
    """精确计算 textbox 实际需要高度"""
    if not text or not text.strip():
        return 0.0
    lines = sum(1 for p in text.split('\n') if p.strip())
    line_h_pt = font_pt * line_spacing
    return lines * line_h_pt / 72.0 + padding * 2
```

**判定**：`box_h < need_h * 0.99` → 压扁 → 必修复。

## 修复原则

| 项 | 之前 | v3.7.5 |
|----|------|-------|
| 主标题 | h=0.50 | **h=1.15**（容纳 22pt × 2 段）|
| 副标题 | h=0.30 | **h=0.40**（容纳 11pt × 1 行）|
| 标题带 | h=0.40 | **h=0.40-0.50**（容纳 13-14pt）|
| 子标题 | h=0.30 | **h=0.30**（12pt × 1.2 = 0.20 + 0.10 缓冲）|
| 3 行内容 | h=0.70-0.78 | **h=0.60**（10pt × 3 × 1.2 = 0.50 + 0.10 缓冲）|
| 核心数据 | h=2.45 | **h=2.20**（4 项 × 0.42）|
| 底部引言 | h=0.25 | **h=0.40**（10pt × 1 行）|
| 卡片标签 | h=0.20 | **h=0.22**（11pt × 1.2 = 0.18 + 0.04 缓冲）|
| 卡片值 | h=0.16 | **h=0.18**（9pt × 1.2 = 0.15 + 0.03 缓冲）|
| 卡片 ext | h=0.16 | **h=0.16**（8pt × 1.2 = 0.13 + 0.03 缓冲）|

```python
# 1. 压扁检查

        if box_h < need_h: FAIL

# 2. 矩形重叠检查
for box1, box2 in combinations(boxes, 2):
    if rect_intersect_area > 0.005: FAIL
```

**目标**：**0 压扁 + 0 重叠 = 真正的零堆叠**。

# v3.8.1 图内文字 = 主题文字

## 核心规则

**放缩图片除了不影响图片的比例，还要保证图片中的文字和主题文字大小差不多**

- 图内文字有效字号 ≈ 主题文字 9-12pt（差 ≤ 2pt）
- 不要让图内 18pt 文字缩到 4pt（看不清）

## 实现思路：2 步法

### Step 1：设计原图时控画布 + 字号

| 要素 | 建议值 | 原因 |
|------|--------|------|
| **画布宽** | 8-10 寸（@ 100dpi = 800-1000px）| 太大缩放后文字小 |
| **原图字号** | 16-20pt | 缩到 4-5 寸宽后 ≈ 8-10pt |
| **scale** | 2x（高分辨 retina）| 实际 px 翻倍 |

### Step 2：显示时按需缩

```python
# plotly 默认 figure 实际原图宽 iw / 100 寸
ref_w_inch = iw_px / 100.0
# 显示 4.0 寸宽,缩放 0.5×
scale = display_w_inch / ref_w_inch
effective_pt = original_pt × scale
# 目标: effective_pt ≈ 9-12pt
```

## 实用公式

| 显示宽 | 原图宽 | 缩放 | 18pt 文字 → 显示 |
|-------|--------|------|-----------------|
| 4.0 寸 | 8.0 寸 | 0.50× | **9pt** ✓ |
| 4.5 寸 | 10.0 寸 | 0.45× | **8.1pt** ✓ |
| 5.0 寸 | 10.0 寸 | 0.50× | 9pt ✓ |
| 5.4 寸 | 18.0 寸 | 0.30× | **4.2pt** ❌ |

**反例**：原图 18 寸宽 + 显示 5.4 寸 = 文字缩到 4pt，**永远要避免**。

# v3.8.2 视觉填满 + 文字框紧凑

## 核心规则

```python
    """设计阶段算出重叠"""
    overlaps = []
    for i in range(len(boxes)):
        for j in range(i+1, len(boxes)):
            ai, ax, ay, aw, ah, az = boxes[i]
            bi, bx, by, bw, bh, bz = boxes[j]
            if az != bz:

            if x_overlap > 0.1 and y_overlap > min_gap:
                overlaps.append((ai, bi, y_overlap))
    return overlaps
```

```python
def check_will_wrap(text, font_pt, box_w):
    """微软雅黑实测系数:
    - 中文字符: 0.014" × pt
    - 英文字母/数字: 0.007" × pt
    - 标点/空格: 0.005" × pt
    - 边界: 预留 5% 缓冲

    cn = sum(1 for c in text if '一' <= c <= '鿿')
```

# v3.8.5v3 — 换行驱动布局 + 缓冲压紧

## 老大原话（2026-06-30 22:50）

> "标题部分做的很好，但是缩短字体之后，大概可以确定不换行。既然不换行了，标题'云端BMS:'和正文'发布：王江海'之间就不要留那么大空间。建议对标题做文字缩减，保证不换行。然后实现换行估计，如果可能换行，就要预留两行的空间。如果不换行就保证信息密度。对于正文也要全部做换行导致的视觉叠加处理。"

## 核心规则（v3.8.5v3）

### 1. 换行检测先于布局

```python
def will_wrap(text, font_pt, box_w):
    """检测 box 宽 + 字号下, 文本会不会换行
    微软雅黑实测: 中 0.014"/pt, 英 0.007"/pt, 标点 0.005"/pt
    """
    cn = sum(1 for c in text if '一' <= c <= '鿿')
    en = sum(1 for c in text if c.isalnum() and not ('一' <= c <= '鿿'))
    sp = len(text) - cn - en
    est_w = cn * 0.014 * font_pt + en * 0.007 * font_pt + sp * 0.005 * font_pt
    eff = box_w - 0.04
    return est_w > eff * 0.95, est_w, eff

def actual_lines(text, font_pt, box_w):
    """实际行数 = 不换行=段落数, 换行=ceil(est_w / eff)"""
    will, est_w, eff = will_wrap(text, font_pt, box_w)
    if not will: return sum(1 for p in text.split("\n") if p.strip())
    return max(1, int(est_w / eff) + 1)
```

### 2. 缓冲压紧规则（老大钦定）

| 区段 | v3.8.5 (旧) | v3.8.5v3 (新) |
|------|------------|--------------|
| **标题不换行 → 跟正文** | 0.30" 浪费 | **0.05"** 压紧 |
| **段落同字号** | 0.30" 浪费 | **0.05-0.10"** 紧贴 |
| **段间异字号** | 0.30" 浪费 | **0.20"** 适中 |
| **可能换行 → 后续** | 不预留 | **预留 2 行空间** |
| **left_band → left_col** | 0.20" 浪费 | **0.05-0.10"** 带状紧贴 |

**核心**: 不换行 = 0.05-0.10" 缓冲; 换行 = 预留 2 行 (lines × font_pt × 1.20 / 72 + 0.10)

### 3. 所有正文做换行检测

## 教训

| 教训 | 细节 |
|------|------|
| **"不换行就不浪费"** | 标题已确认不换行, 缓冲 0.30" → 压到 0.05" |
| **"换行检测先于布局"** | 设计前检测, 换行的段预留 2 行 h, 不换行压紧 |
| **"每段都做检测"** | 副标题 5 段, 正文每行都检测, 换行预留 |
| **"压紧但不撞"** | 0.05-0.10" 紧贴 OK, 0.00 以下 = 撞 |
| **"带状装饰可紧贴"** | left_band + left_col 0.04-0.10" 是标准模式 |
| **"段间异字号 0.20"** | 字号差 ≥ 2pt 留 0.20", 避免视觉粘连 |
| **"缓冲 = 0 时一定撞"** | 0.00" 缓冲 = textbox 实际贴边, 渲染时字体溢出 0.05" 必撞 |
| **"老大截图是终极标准"** | 算法对 ≠ 视觉对, 必须压缩缓冲后让老大再验 |
