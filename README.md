## 它解决什么问题？

大多数 AI 生成的幻灯片看起来很"通用"——字体不对、颜色不搭、到处是水印。

**Aily PPT 模板生成器**走的是另一条路：

1. **你提供**公司现有的 PPT 模板（1–5 页同版式示例）
2. **它学习**模板的"DNA"——哪些文字是标签（保留），哪些是变量（替换），zone 怎么划分
3. **它生成**新页面——视觉上和你模板一模一样，但填充了新内容

**零水印。100% 可在 PowerPoint 中编辑。像素级模板还原。**

## 核心能力

### 模板智能
- **多页归纳** — 从 3–5 张同版式示例中，统计归纳共性 zone（机构标签区/主标题区/正文区/出处区），不靠硬编码规则
- **Run 级 slot 检测** — 在每个文本 run 的粒度上，区分"模板"run（机构标签、章节标记）和"变量"run（需要替换的内容）
- **半结构化识别** — 同一段落中既有固定结构文字也有动态内容时，也能正确处理

### 布局引擎
- **14 条布局原则（P0–P3 优先级）** — 经过实战验证的规则体系，覆盖零堆叠、色彩纪律、信息密度、视觉填满、自适应间距
- **堆叠防护（P0）** — 基于真实行高计算（`will_wrap()` + `actual_lines()` + `calc_h_auto()`），针对微软雅黑及常见中文字体校准
- **自适应间距** — 按内容驱动 zone 尺寸（"先看菜点菜，再切锅"），而非固定网格
- **信息密度强制** — 每页最低 1,000–1,800 字符，7 类信息类型齐全（数字、表格、加粗关键词、引用、金句、图示、行业数据）

### 输出质量
- **无水印** — 直接通过 `python-pptx` 操作原始模板
- **完全可编辑** — 输出标准 `.pptx`，可在 PowerPoint / Keynote / Google Slides 中打开编辑
- **多页批量生成** — 一次生成 N 页，结构一致
- **样式继承** — 字体、颜色、字号、加粗/斜体、装饰元素全部从模板继承

## 架构：三层抽象

```
┌─────────────────────────────────────────────────┐
│  第 1 层：版式层 (Layout)                         │
│  模板的视觉 DNA：字体、颜色、位置、装饰            │
├─────────────────────────────────────────────────┤
│  第 2 层：特征层 (Features)                       │
│  从 N 张示例 slide 归纳出的 zone 结构             │
├─────────────────────────────────────────────────┤
│  第 3 层：内容层 (Content)                        │
│  用户按 zone × slide 提供的新填充数据             │
└─────────────────────────────────────────────────┘
```

## 流水线

```
模板 .pptx（N 页）
    │
    ▼
extract_template_features.py  →  template_features.json
    │                              （归纳 zone：机构/标题/正文/出处）
    ▼
detect_template_slots.py      →  slot_map.json
    │                              （run 级：模板 vs 变量）
    ▼
AI 生成 content.json          →  每页的 fill_map
    │
    ▼
generate_pptx.py              →  output.pptx（多页、可编辑）
```

## 布局原则（v3.8.5v3）

引擎强制执行 **14 条原则**，按优先级排列：

| 优先级 | 原则 | 规则 |
|--------|------|------|
| **P0** | 零堆叠 | 绝对不允许文字/形状碰撞——真实行高数学，不是包围盒猜测 |
| **P1** | 三色制 | 每页 ≤ 3 种主色（如深蓝/红/深灰）|
| **P1** | 图下表上 | 图表标题放在图/表**下方**，纯黑 #000000 |
| **P1** | 图内文字 = 主题文字 | 缩放图片使图内文字 ≈ 正文字号（9–12pt 有效）|
| **P2** | 视觉填满 | 文字填满页面——不挂一隅，双栏均衡 |
| **P2** | 空间自适应 | zone 尺寸跟随内容，不跟固定网格 |
| **P3** | 信息密度 | 1,000–1,800 字/页，7 类信息类型齐全 |
| **P3** | 多页归纳 | 从 N 张示例统计提取 zone |
| **P3** | Run 级分类 | 模板 vs 变量精确到文本 run 粒度 |
| **P3** | 缓冲 ≥ 0" | 不允许负间距；0.05–0.10" 紧贴 OK |
| **P3** | 换行检测先于布局 | 布局决策前先调 `will_wrap()` |
| **P3** | 标题 ≤ 14 字 | 18–20pt 下 7.7" 宽 box 内不换行 |

**冲突解决**：P0 > P1 > P2 > P3。原则冲突时，高优先级胜出。密度可让步到 1,000+；堆叠绝不让步。

## 使用方法

```bash
# 第 1 步：从 N 张示例中提取模板特征
python scripts/extract_template_features.py template.pptx features.json

# 第 2 步：检测模板 vs 变量 slot
python scripts/detect_template_slots.py template.pptx slots.json

# 第 3 步：AI 生成 content.json（每页的 fill_map）
# 通常由 AI Agent（如 Aily）处理

# 第 4 步：生成最终 .pptx
python scripts/generate_pptx.py template.pptx slots.json content.json output.pptx
```

## 设计理念

> **"模板与内容分离。从示例学习结构。Run 级精度替换。"**

1. **三层抽象分离** — 版式 / 特征 / 内容是独立关注点
2. **统计归纳优于规则** — 从 N 张示例学习 zone 模式，不靠硬编码 if-else
3. **样式继承** — 永不硬编码颜色/字体；模板的视觉 DNA 自动流转
4. **Run 级精度** — 每个文本 run 单独分类
5. **可编辑输出** — 用户拥有最终 `.pptx`，无锁定
6. **零水印** — 干净输出，永远
